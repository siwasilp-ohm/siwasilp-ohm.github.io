การวิเคราะห์ระบบ CCD Platform

เอกสารนี้จัดทำขึ้นเพื่อนำเสนอภาพรวมเชิงลึกของสถาปัตยกรรมและฟังก์ชันการทำงานหลักของแพลตฟอร์ม CCD (Continuing Competency Development & University-Social Engagements) เพื่อให้ท่านลูกค้าเข้าใจถึงองค์ประกอบต่างๆ และศักยภาพในการพัฒนาต่อยอด

ส่วนประกอบของระบบ (System Components)

  • ส่วนหน้าบ้าน (Frontend): พัฒนาด้วย HTML, Tailwind CSS และ JavaScript เพื่อสร้างส่วนต่อประสานผู้ใช้ (UI) ที่ตอบสนอง (Responsive) และสวยงามเหมือนแอปพลิเคชันมือถือ พร้อมแอนิเมชันที่น่าดึงดูด
  • ส่วนหลังบ้าน (Backend - จำลอง): หากพัฒนาจริง จะใช้ Node.js ร่วมกับ Express.js เพื่อจัดการตรรกะทางธุรกิจ (Business Logic), การจัดการผู้ใช้ (User Management), การจัดการข้อมูล (Data Management) และการให้บริการ API แก่ส่วนหน้าบ้าน
  • ฐานข้อมูล (Database - จำลอง): แนะนำให้ใช้ Firestore (NoSQL Database) สำหรับการจัดเก็บข้อมูลแบบเรียลไทม์ (Real-time), การปรับขนาดได้ (Scalability) และความยืดหยุ่นในการจัดเก็บข้อมูลประเภทต่างๆ

ความสามารถหลัก (Key Functionalities)

  • การเข้าสู่ระบบ/ลงทะเบียน: รองรับการเข้าสู่ระบบด้วยบัญชีโซเชียลมีเดีย
  • ศูนย์ประกาศ (Dashboard): แสดงภาพรวมและเมนูนำทางไปยังฟังก์ชันหลัก
  • การจัดการบริการ: โพสต์, ดูรายการ, กรอง, และจัดการสถานะบริการ (เสนอ, ได้รับเชิญ, หมดอายุ)
  • การวิเคราะห์ข้อมูล/รายงาน: แสดงสถิติและรายงานประสิทธิภาพแพลตฟอร์ม
  • การจับคู่เพื่อน: ค้นหาและเชื่อมต่อกับผู้เชี่ยวชาญ/เพื่อนร่วมงาน
  • การจับคู่หลักสูตร: ค้นหา, กรอง, และลงทะเบียนหลักสูตรพัฒนาทักษะ
  • การจับคู่เนื้อหา: สำรวจข่าวสาร, บทความ, อินโฟกราฟิก, และวิดีโอที่เกี่ยวข้อง
  • การจับคู่ทางสังคม: โพสต์ข้อความ, รูปภาพ, วิดีโอ และโต้ตอบในฟีดกิจกรรม
  • การจับคู่รางวัล: แลกคะแนนสะสมเป็นรางวัลต่างๆ
  • การตั้งค่าโปรไฟล์และบัญชี: จัดการข้อมูลส่วนตัว, ความปลอดภัย, การแจ้งเตือน, และการตั้งค่าแอปพลิเคชัน

โครงสร้างฐานข้อมูล (Conceptual Database Schema)

นี่คือโครงสร้างข้อมูลหลักที่เสนอสำหรับการจัดเก็บในฐานข้อมูล Firestore:

Users Collection

  • userId: String (Unique ID)
  • name: String
  • email: String
  • profilePictureUrl: String (URL ไปยังรูปภาพ)
  • points: Number (คะแนนสะสม)
  • role: String (Admin, Learner, Coach, Distributor)
  • phone: String (Optional)
  • address: String (Optional)
  • settings: Map (Nested object for user preferences)
    • notifications: Map (newActivity, messages, emailNotif - Boolean)
    • privacy: Map (profileVisibility, friendRequest - String)
    • app: Map (language, theme - String)
    • security: Map (twoFactorAuth - Boolean)

Services Collection

  • serviceId: String (Unique ID)
  • title: String
  • detail: String
  • group: String (e.g., Equip&Test, Consult, Training)
  • owner: String (ชื่อเจ้าของบริการ)
  • distributor: String (ชื่อผู้จัดจำหน่าย)
  • expense: String (ช่วงราคาหรือ "ต่อรองได้")
  • dueDate: Date (วันครบกำหนด)
  • postedDate: Date (วันที่โพสต์)
  • durationDays: Number (ระยะเวลาเป็นวัน)
  • contact: String (เบอร์ติดต่อ)
  • customerGroup: String (e.g., ทั่วไป, บุคคล, องค์กร)
  • status: String (e.g., เปิด, ปิด, หมดอายุ)
  • type: String (Recommend, Offer, Invited, Keep)
  • imageUrl: String (URL ไปยังรูปภาพบริการ)

Courses Collection

  • courseId: String (Unique ID)
  • title: String
  • detail: String
  • instructor: String
  • duration: String (e.g., "8 สัปดาห์")
  • rating: Number (คะแนนเฉลี่ย)
  • reviews: Number (จำนวนรีวิว)
  • category: String (e.g., เทคโนโลยี, การจัดการ, ภาษา)
  • imageUrl: String (URL ไปยังรูปภาพหลักสูตร)

Content Collection

  • contentId: String (Unique ID)
  • title: String
  • detail: String (สรุปย่อ)
  • type: String (ข่าวสาร, บทความวิชาการ, อินโฟกราฟิก, วิดีโอ)
  • publishDate: Date
  • views: Number
  • imageUrl: String (URL ไปยังรูปภาพเนื้อหา)

SocialPosts Collection

  • postId: String (Unique ID)
  • userId: String (อ้างอิงถึง Users Collection)
  • postText: String
  • imageUrl: String (Optional, URL)
  • videoUrl: String (Optional, URL)
  • timestamp: Timestamp (วันที่และเวลาโพสต์)
  • likesCount: Number
  • commentsCount: Number
  • sharesCount: Number

Rewards Collection

  • rewardId: String (Unique ID)
  • name: String
  • description: String
  • pointsRequired: Number
  • imageUrl: String (URL ไปยังรูปภาพรางวัล)
  • status: String (Available, Redeemed)

UserRewards Collection

  • userRewardId: String (Unique ID)
  • userId: String (อ้างอิงถึง Users Collection)
  • rewardId: String (อ้างอิงถึง Rewards Collection)
  • redeemedDate: Timestamp
  • status: String (Pending, Completed, Cancelled)
